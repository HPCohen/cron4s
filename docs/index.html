First Steps

To start using cron4s in your project just include the library as part of your dependencies:

libraryDependencies += "com.github.alonsodomin.cron4s" %% "cron4s" % "x.y.z"

Or in ScalaJS:

libraryDependencies += "com.github.alonsodomin.cron4s" %%% "cron4s" % "x.y.z"

Parsing

cron4s uses cron expressions that go from seconds to day of week in the following order:

  • Seconds
  • Minutes
  • Hour Of Day
  • Day Of Month
  • Month
  • Day Of Week

To parse a cron expression into a type that we can work with we will use the Cron smart constructor:

scala> import cron4s._
import cron4s._

scala> val parsed = Cron("10-35 2,4,6 * * * *")
parsed: Either[cron4s.InvalidCron,cron4s.expr.CronExpr] = Right(10-35 2,4,6 * * * *)

We will get an Either[InvalidCron, CronExpr], the right side giving us an error description if the parsing has failed. In the above example the expression parsed successfully but if we pass a wrong or invalid expression we will get the actual reason for the failure in the left side of the Either:

scala> val invalid = Cron("10-65 * * * * *")
invalid: Either[cron4s.InvalidCron,cron4s.expr.CronExpr] = Left(ParseFailed(" ":1:5 ..."5 * * * * ",4))

Assuming that we have successfully parsed an expression, we can extract it out of the Either[..., ...] with following expression:

scala> val cron = parsed.right.get
cron: cron4s.expr.CronExpr = 10-35 2,4,6 * * * *

Note: It is not recommended to use .right.get to extract values out of Either[..., ...] types, we are doing it here as a means of simplifying the types just for the sake of the tutorial.

Validation errors

The CRON expression will be validated right after parsing all any error found during this stage will be returned as well in the left side of the Either returned by the Cron constructor. For instance, the following expression has a sequence that mixes elements that self-imply each other, which is invalid:

scala> val invalid = Cron("12,10-20 * * * * *")
invalid: Either[cron4s.InvalidCron,cron4s.expr.CronExpr] = Left(ValidationError(NonEmpty[FieldError(Second,Value '12' at field Second is implied by '10-20')]))

The Cron4s AST

After successfully parsing a CRON expression, the CronExpr resulting type represents the previously parsed expression as an AST, in which we can access all the expression fields individually:

scala> cron.seconds
res2: cron4s.expr.SecondsNode = 10-35

scala> cron.minutes
res3: cron4s.expr.MinutesNode = 2,4,6

scala> cron.months
res4: cron4s.expr.MonthsNode = *

We can also take the date or time parts only of the expression using either timePart or datePart:

scala> val time = cron.timePart
time: cron4s.expr.TimeCronExpr = 10-35 2,4,6 *

scala> val date = cron.datePart
date: cron4s.expr.DateCronExpr = * * *

And similarly as with the main expression type, we can get individual fields out of the date and time sub expressions:

scala> time.seconds
res5: cron4s.expr.SecondsNode = 10-35

scala> time.minutes
res6: cron4s.expr.MinutesNode = 2,4,6

scala> date.daysOfMonth
res7: cron4s.expr.DaysOfMonthNode = *

Or by means of the field method in CronExpr and passing either the cron field type or the cron unit.

scala> cron.field[CronField.Minute]
res8: Option[cron4s.expr.FieldNode[cron4s.CronField.Minute]] = Some(2,4,6)

scala> cron.field(CronUnit.Minutes)
res9: Option[cron4s.expr.FieldNode[cron4s.CronField.Minute]] = Some(2,4,6)

Some other basic operations at the CronExpr level are asking for the list of supported fields of the actual value ranges for all the fields in the form of a map:

scala> cron.supportedFields
res10: List[cron4s.CronField] = List(Second, Minute, Hour, DayOfMonth, Month, DayOfWeek)

scala> cron.ranges
res11: Map[cron4s.CronField,IndexedSeq[Int]] = Map(Second -> Range 10 to 35, DayOfMonth -> Range 1 to 31, Month -> Range 1 to 12, DayOfWeek -> Range 0 to 6, Hour -> Range 0 to 23, Minute -> Vector(2, 4, 6))

To convert an AST back into the original string expression we simply use the toString method:

scala> cron.toString
res12: String = 10-35 2,4,6 * * * *

Sub-expressions

All the operations possible on a CronExpr are also possible in any of its subexpressions (either time or date) so you can use them in exactly the same way. For example:

scala> cron.supportedFields
res13: List[cron4s.CronField] = List(Second, Minute, Hour, DayOfMonth, Month, DayOfWeek)

scala> cron.timePart.supportedFields
res14: List[cron4s.CronField] = List(Second, Minute, Hour)

scala> cron.datePart.supportedFields
res15: List[cron4s.CronField] = List(DayOfMonth, Month, DayOfWeek)

supportedFields is not super-interesting at CronExpr (we expect it to support all the fields anyway) but when is part of the sub-expressions gives us a more particular piece of information about the actual expression itself. The field method is also interesting and justifies its signature when used with the sub-expressions:

scala> cron.timePart.field(CronUnit.Seconds)
res16: Option[cron4s.expr.FieldNode[cron4s.CronField.Second]] = Some(10-35)

scala> cron.timePart.field[CronField.DayOfMonth]
res17: Option[cron4s.expr.FieldNode[cron4s.CronField.DayOfMonth]] = None

scala> cron.datePart.field(CronUnit.Seconds)
res18: Option[cron4s.expr.FieldNode[cron4s.CronField.Second]] = None

scala> cron.datePart.field[CronField.DayOfWeek]
res19: Option[cron4s.expr.FieldNode[cron4s.CronField.DayOfWeek]] = Some(*)

This is just a teaser, we will see much more interesting operations on cron expressions later but it’s good to know that all operations possible on a CronExpr, are also possible on it’s subexpressions.

Field nodes

All field nodes have their own type, which is parameterized in the actual field type they operate on. We can access that field type definition via the unit of field expression:

scala> cron.seconds.unit.field
res20: cron4s.CronField.Second = Second

The expression unit can be used to give us information about what values are valid for that specific field:

scala> cron.seconds.unit.range
res21: IndexedSeq[Int] = Range 0 to 59

Which is different than the range of values accepted by the expression at that given field:

scala> cron.seconds.range
res22: IndexedSeq[Int] = Range 10 to 35

We can also obtain a field expression

To obtain the string representation of individual fields we use the same toString method:

scala> cron.seconds.toString
res23: String = 10-35

scala> cron.field[CronField.Minute].toString
res24: String = Some(2,4,6)

Other interesting operations are the ones that can be used to test if a given value matches the field expression:

scala> cron.seconds.matches(5)
res25: Boolean = false

scala> cron.seconds.matches(15)
res26: Boolean = true

scala> cron.minutes.matches(4)
res27: Boolean = true

scala> cron.minutes.matches(5)
res28: Boolean = false

Or to test if a given field expression is implied by another one (that is also parameterized by the same field type). To show this, let’s work with some simple field expressions:

scala> import cron4s.expr._
import cron4s.expr._

scala> val eachSecond = EachNode[CronField.Second]
eachSecond: cron4s.expr.EachNode[cron4s.CronField.Second] = *

scala> val fixedSecond = ConstNode[CronField.Second](30)
fixedSecond: cron4s.expr.ConstNode[cron4s.CronField.Second] = 30

scala> fixedSecond.implies(eachSecond)
res29: Boolean = false

scala> fixedSecond.impliedBy(eachSecond)
res30: Boolean = true

scala> eachSecond.implies(fixedSecond)
res31: Boolean = true

scala> val minutesRange = BetweenNode[CronField.Minute](ConstNode(2), ConstNode(10))
minutesRange: cron4s.expr.BetweenNode[cron4s.CronField.Minute] = 2-10

scala> val fixedMinute = ConstNode[CronField.Minute](7)
fixedMinute: cron4s.expr.ConstNode[cron4s.CronField.Minute] = 7

scala> fixedMinute.implies(minutesRange)
res32: Boolean = false

scala> fixedMinute.impliedBy(minutesRange)
res33: Boolean = true

These two operations allways hold the following property (it looks obvious, but it’s important):

scala> assert(minutesRange.implies(fixedMinute) == fixedMinute.impliedBy(minutesRange))

It’s important to notice that when using either the implies or impliedBy operation, if the two nodes are not parameterized for the same field type, the code won’t compile:

scala> minutesRange.implies(eachSecond)
<console>:21: error: no type parameters for method implies: (ee: EE[cron4s.CronField.Minute])(implicit EE: cron4s.expr.FieldExpr[EE,cron4s.CronField.Minute])Boolean exist so that it can be applied to arguments (cron4s.expr.EachNode[cron4s.CronField.Second])
 --- because ---
argument expression's type is not compatible with formal parameter type;
 found   : cron4s.expr.EachNode[cron4s.CronField.Second]
 required: ?EE[cron4s.CronField.Minute]
       minutesRange.implies(eachSecond)
                    ^
<console>:21: error: type mismatch;
 found   : cron4s.expr.EachNode[cron4s.CronField.Second]
 required: EE[cron4s.CronField.Minute]
       minutesRange.implies(eachSecond)
                            ^
<console>:21: error: could not find implicit value for parameter EE: cron4s.expr.FieldExpr[EE,cron4s.CronField.Minute]
       minutesRange.implies(eachSecond)
                           ^

The error looks a bit scary, but in essence is saying to us that the implies method was expecting any kind of expression as long as it was for the Minute field (expressed as EE[cron4s.CronField.Minute]).
